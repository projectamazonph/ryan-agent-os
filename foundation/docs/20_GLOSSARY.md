# Glossary

**Agent definition**: A versioned specification of an agent's purpose, tools, policies, inputs, outputs, and evaluation rubric.

**Agent run**: One execution attempt by an agent for a specific task.

**Approval gate**: A policy-enforced pause before an action that can affect an external system, another person, money, production, or sensitive data.

**Artifact**: A concrete output such as a document, spreadsheet, slide deck, code change, report, or image.

**Capture**: Raw incoming context submitted to Ryan Agent OS.

**Context package**: The bounded set of task-relevant information provided to an agent.

**Decision**: A durable record of a choice, rationale, alternatives, and impact.

**Deliverable**: A required project output with acceptance criteria and a destination.

**Evidence**: Verifiable proof that work satisfies an acceptance criterion.

**Execution pack**: A versioned plan containing scope, deliverables, tasks, dependencies, risks, decisions, approvals, and acceptance criteria.

**Implementation queue**: The prioritized cross-project view of actionable work.

**Memory item**: Curated reusable context with source, scope, confidence, and lifecycle.

**Model gateway**: The internal abstraction that routes requests to local or cloud model providers.

**Project**: The durable unit representing an intended outcome and its coordinated work.

**Protected action**: An action that requires explicit approval under policy.

**Provenance**: The trace linking an output to its source inputs, producer, model, prompt, tools, and versions.

**Task graph**: Tasks and their dependency relationships.

**Verification**: Evidence-based confirmation that acceptance criteria were met.
