# Contributing

## Contribution flow

1. Start from an approved backlog item.
2. Confirm affected requirements and ADRs.
3. Make the smallest coherent change.
4. Add or update tests.
5. Update documentation in the same change.
6. Attach verification evidence.
7. Open a reviewable pull request.

## Pull request expectations

A pull request must state:

- Problem
- Scope
- Design choice
- Tests and evidence
- Security impact
- Documentation changes
- Known limitations

Do not mix unrelated refactors with a feature or fix unless the dependency is unavoidable and explained.
